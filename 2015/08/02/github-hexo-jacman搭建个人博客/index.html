博客基础知识

所谓博客，其实就是一个网站，像你打开的淘宝，京东这样的网站一样，只不过博客是一个以文章展示为主的网站。那么，怎样才能让别人通过网络访问自己搭建的网站呢？不急，我们先来想这个问题，我们是以什么方式访问网站的？都知道，是通过在浏览器中输入网址，然后敲回车就好了（不要反驳我说明明很多情况都是点击链接访问的）。好，知道了这个问题，那下一个问题是我们打开一个网页看到的所有的这些内容是放在什么地方的？你看到的网页内容一般是文本、视频、图片等，这些东西一定是放在了某一个地方，要不然我们不可能看到。也就是说，我们需要一个类似于容器一样的东西来”盛放“这些东西。通过上面的分析，我们知道，一个网站想要能被人看到，除了内容本身以外，还需要一个地址，这就是网址；另一个是一个”盛放“内容的容器，那就是空间。

什么是网址

举个例子，app.baidu.com就是一个网址，通过在浏览器地址栏输入此内容，可以访问百度开发者首页。通常完整的有效网址结构如下：

我们应该关心的是域名，因为这个需要购买，当然如果用github的话，就不需要购买了，直接用其提供给用户的二级域名就行了，后面再具体说明。其实这个时候你应该还是有个疑问，为什么有了域名就可以访问我们放在某个地方的内容了？其实，在网络世界里，宏观上，我们在访问一个地方的内容的时候，都是通过IP地址来访问的，形如：115.239.211.221。每一台联网的电脑都有一个IP地址，它是一台主机在网络中的唯一标识符，就像一个人的DNA一样。我们想访问一个网站，如果记忆IP地址，显然是不可能的（这乱七八糟的数字谁能记住）。如果有一种方便记忆的东西代替IP地址（与IP地址一一对应），那么是不是很爽？这就是域名的作用。域名和IP地址之间是一一对应的关系，他们之间有一套解析协议，当你在浏览器的地址栏中输入域名地址时，它会被解析成对应的IP地址，然后访问IP地址指向的位置，至于是如何解析的，我们不用关心。到目前为止，我们访问一个网站的过程和方式似乎能想得明白了，但是还有一个问题没有说明白，那就是那个“盛放”内容的容器，也就是空间到底是什么？

什么是空间

在网络中，所有的文本、图片、视频等资源统统放在一种地方，这种地方实际上就相当于电脑的硬盘一样，它可以存放数据，这个地方就是空间，也即虚拟空间。事实上，这些虚拟空间的载体实际上是一台台装有linux系统的电脑，有硬盘，有处理器等。（当然也不尽然），只不过他不是作为我们平常的学习工作使用的电脑，而是只是提供数据的存储和处理服务。我们网站的内容都是储存在它上面。每一台主机都有一个IP地址，我们通过IP地址就能访问我们存放数据的具体位置。通常情况下，虚拟空间的服务都不是免费提供的，需要我们花钱购买，基本上都是按年付费。

通过以上分析，我们已经知道了，想要搭建一个网站（在我们这里就是博客）需要什么条件，以及具体是怎么一回事。那么，我们需要的域名和空间该从何而来呢？当然是万能的github给我们提供了。当我们注册github之后，github会根据我们注册时的用户名为我们分配一个二级域名，像这样的：suncrazy.github.io。其中，suncrazy也就是你注册时的用户名，所以大家在注册github的时候要想好用户名了。同时，github会为我们免费提供300M的空间，没有使用的限制。简直不能更棒，有木有！！！这样一来，万事具备啊，还等什么，开干？不急不急，我们上面屡次提到github，那么什么是github呢？

Markdown简介

markdown实际上是一种类HTML的写作语言，它可以让写作者把几乎所有的精力放在编辑文档上面，而不用额外添加文档的样式。他是怎么做到的呢？它实际上是html的简化版，通过定义一个个标签，而这些标签的样式是已经定义好了的，当我们在文档中使用这些标签的时候，编辑器会自定把其转化成对应的样式。所以，基本上我们在写完文档内容的时候，样式也就被随之确定了，不需要额外的加工。关于markdown的语法，建议参考markdown简明语法。语法很简单，希望大家认真学习，基本上5分钟就学会啦。
想用markdown写作，必须有一个支持markdown语法的编辑器。现在有很多编辑器都支持markdown语法，同时还支持实时显示（就是左边编辑markdown，右边显示具体的效果）。我试了很多编辑器，发现好用的没多少。在这里我给大家的推荐是：

SSH密钥对

我们已经注册好了github，那么我们怎么才能本机与github之间建立安全的通信呢？毕竟本地编辑的博客内容想要托管到github，必须得和它之间建立通信连接，而且是安全的连接。要不然任何人都可以将内容推送到你的github仓库，那不就乱套了。怎么保证安全性呢？那就是通过SSH。github是通过SSH来识别用户的合法性的。我们必须先在github里保存一份与我们现在使用的电脑有关的SSH秘钥，然后我们在与github建立通信时才能被github有效地识别。

建立博客仓库

文章中如何使用图片

在用markdown写文章的时候，插入图片非常简单，只要有图片的地址就行了。但是，一旦你把博客上传到github，这个图片的位置就不好确定了，你都不知道该把图片放在哪个文件夹里面。有人说，我们可以先在根目录下的“source”目录里建一个文件夹，然后此文件夹里面存放我们文章中使用到的图片。看起来没有问题，但是当你尝试之后你会发现有很大的问题。使用这一种方法，文章在主页中显示图片是完美的，但是一旦进入一篇文章的具体内容页时，你就傻眼了，因为图片显示不出来。为什么呢？你看一下文章的内容页的地址就会发现，你上传的博客会根据分类被放在另一个文件夹里面，这样导致的后果时，你在生成博客之后，文章的位置与图片的位置的相对关系发生了变化，这样就会导致你进入内容页之后，页面根据你之前设置的图片地址无法找到对应的图片。那么，如何解决这个问题呢？我采用的办法是使用一个图片存储服务器。

思路其实很简单。就是，我们先把图片存放在另一个地方，然后该地方告诉我每一张图片的地址（我能通过这个地址唯一地获取到这张图片），然后我在写文章的时候，图片的地址直接设置成这个地址就好了。在这里，我使用的是七牛，它提供免费的1G存储图片和文件的服务，并给每一个文件一个唯一的地址。对于我们个人博客使用者来说，简直不能更赞了。首先，登陆七牛官网,怎么注册和使用可以参考这个文档。

配置独立域名

如果你觉得使用github给你提供的二级域名让人很不爽，想搞一个自己的域名。没关系，花点钱，买一个就好了。怎么购买域名，我也是菜鸟，这是第一次买域名，也是第一次尝试使用。有人建议到godaddy,但是我觉得有点贵，而且麻烦。如果你想在这里买的话，可以看看这篇博客。

我购买域名的地方是新网，首先，你进入网站之后点击“首页”，然后搜索一下你想要的域名，一方面是看看这个域名有没有被别人注册，另一方面看看价格。你要注意的是，它上面标记的价格是首年（第一年）的费用，第二年是多少你得多注意。一般首年的域名费用都很便宜。选好域名之后，就是购买了，都是正常的程序，先注册账号，然后购买（与网购一样）。付款成功之后，剩下的就是配置了。

先将鼠标放在你的邮箱上面，也就是购物车左边的那一个区域，如图：

点击”进入我的账户“则可进入账户的配置页面。然后点击左边的”我的产品“，在“常规管理”里面找到“域名管理”，然后点击进入。点击“我的域名”即可看到我们购买的域名。

选中我们购买的域名，然后点击列表下方的“mydns解析设置”,进入如下页面：

在”别名（CNAME）“这一项里面，点击”添加新的别名“，在出现的第一的输入框里面填入”www“,或者你想要的任意字母。以后你就是通过”XXXX.域名”来访问你的博客了。然后在第二个输入框里面输入你的博客的github二级域名，例如”username.github.io”，然后保存。这样，域名的解析设置就完成了。

最后需要做的是让github知道每次只要有人访问你的博客的时候，直接将地址指向你的域名处。怎么设置呢？打开博客的根目录，然后在”source“目录下新建一个名字为”CNAME“的文件（注意，文件的全名就是CNAME，没有后缀）。然后用你的编辑器打开CNAME文件，在里面添加一行字母。就是你在”mydns设置“里面设置的别名，例如我设置的”www.tortman.top“,我就在此文件里面写入www.tortman.top就行啦。完事之后，重新将博客上传github。

到目前为止，我们就配置好了独立的域名，过一段时间就可以通过你自己的域名访问你的博客了。

总结

本篇文章都是以github为载体进行讲解的。其实你也可以用gitcafe，它与github的区别是，gitcafe是国内的网站，访问速度会快一点，不存在被墙掉问题。但是，gitcafe有个问题是最近经常出问题，老是被攻击瘫痪。不管怎么样，你都可以去尝试做做。甚至两个同时使用，很简单就可以实现。

其实，建立自己的博客也是一个学习的过程，大家在学习的过程中，一定要有耐心。遇到问题，要自己努力查资料解决问题，一定会有解决的办法的。
